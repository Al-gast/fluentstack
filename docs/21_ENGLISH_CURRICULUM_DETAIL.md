# English for Tech Careers Curriculum Detail

## Purpose

This document defines the detailed English for Tech Careers curriculum for FluentStack.

It guides future content batches from basic work-English foundations to professional remote-work English fluency for tech careers. It is not a generic grammar course and it is not raw lesson content.

English fluency in FluentStack does not mean native-speaker perfection. It means the learner can work remotely in tech using English clearly and confidently.

Actual lesson content inside the product should use a bilingual format:

- Indonesian explanation.
- Practical English examples.
- Indonesian meaning or context when useful.
- Learner writes their own English response.

Common tech terms may stay in English when natural: `daily update`, `blocker`, `pull request`, `bug report`, `code review`, `deployment`, `API`, `frontend`, and similar terms.

## Track Outcome

A learner who completes this track should be able to:

- Write daily updates, blockers, bug reports, and PR descriptions.
- Ask clarification before building the wrong thing.
- Explain technical work, trade-offs, and risks in clear English.
- Participate in code review and async remote collaboration.
- Present portfolio projects and technical decisions in interviews.
- Maintain a practical English routine without chasing native-speaker perfection.

## Activity Type Reference

Use these activity types across modules:

- `text`: short explanation of one work situation or language pattern.
- `callout`: common mistake, tone note, phrase rule, or workplace context.
- `quick-check`: choose the clearer sentence, rewrite a phrase, or identify missing context.
- `quiz`: concept check for structure, tone, and meaning.
- `writing-practice`: learner writes their own update, note, report, answer, or script.
- `case study`: inspect a realistic workplace message and improve it.
- `self-review checklist`: check clarity, tone, specificity, and next action.
- `speaking script practice`: prepare short spoken answers for meetings, demos, or interviews.

## Relationship With Frontend Engineering Track

English lessons should support technical lessons instead of becoming a separate generic English course.

Recommended connection pattern:

1. Frontend lesson teaches the technical skill.
2. English lesson teaches how to communicate the same work situation.
3. Writing or speaking practice uses the same project context when possible.
4. The output becomes a realistic work artifact: PR note, blocker update, bug report, review comment, README, demo script, or interview answer.

Examples:

| Frontend Engineering context | English for Tech Careers support |
| --- | --- |
| Build login form | Write a pull request note explaining validation, edge cases, and test steps. |
| Handle API errors | Explain a blocker caused by an unexpected API response. |
| Accessibility audit | Write a code review comment about keyboard navigation. |
| Responsive navbar | Write a daily update about layout changes and remaining review items. |
| Supabase Auth integration | Write a handoff note explaining session behavior and known limitations. |
| Portfolio project | Present the project in English with problem, solution, features, trade-offs, and next steps. |

## Content That Should Wait

Wait until basic remote communication is stable before creating:

- Advanced stakeholder updates.
- Incident communication.
- Migration notes.
- Mentoring messages.
- Salary negotiation deep dives.
- Long interview simulations.
- Complex architecture decision records.
- Advanced async decision-making scenarios.

Reason:

- These topics require confidence with basic updates, blockers, clarification, and PR notes.
- They are more useful when learners already have frontend projects to discuss.
- They should reuse realistic project context instead of becoming abstract language drills.

## Authoring Notes for Future English Content

- Keep explanations in Bahasa Indonesia.
- Use English examples that sound natural in remote tech work.
- Add Indonesian meaning or context only when it helps the learner understand tone or usage.
- Ask learners to write their own English response in every writing-focused lesson.
- Avoid grammar-first lesson titles unless the grammar directly solves a work communication problem.
- Keep feedback calm and specific.
- Prefer realistic artifacts: daily update, blocker note, bug report, PR note, review comment, README, demo script.
- Connect English practice to Frontend Engineering lessons whenever possible.
