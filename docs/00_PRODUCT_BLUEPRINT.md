# FluentStack Product Blueprint

## Product Name

FluentStack

## One-Line Description

FluentStack is an interactive learning platform for technology skills, combining structured lessons, practice blocks, quizzes, coding labs, writing exercises, and progress tracking.

## Initial Tracks

1. Frontend Engineering
2. English for Tech Careers

## Future Tracks

* Backend Engineering
* Computer Science Fundamentals
* Databases
* Cloud Fundamentals
* AI for Developers
* System Design
* Cybersecurity Basics
* Career Preparation

## Target Users

Beginner to intermediate learners who want to build practical technology skills and become ready for real-world work, including remote opportunities.

## Main Problem

Most learners consume too much passive content.

They watch videos, read articles, or follow tutorials, but they often do not practice enough, do not track progress clearly, and do not build strong proof of skill.

Many learners also separate technical learning from communication skills, even though remote tech work requires both.

## Product Solution

FluentStack turns learning into an active flow.

Users do not just read lessons. They also:

* answer quick checks
* complete quizzes
* write short responses
* practice coding in the browser
* build small projects
* track progress
* follow a structured roadmap

## Product Positioning

FluentStack should feel like a guided learning lab for technology skills.

It should combine:

* structured roadmap
* clear lessons
* active practice
* useful feedback
* visible progress
* polished product experience

The product should be useful for real learning and strong enough to be shown as a portfolio project.

## Core Product Principles

### 1. Learning must be active

Every important lesson should include interaction when useful.

Examples:

* quiz
* coding practice
* writing practice
* checklist
* reflection prompt

### 2. Practice belongs inside the lesson

Quiz and practice are not separate disconnected pages.

They are part of the lesson flow.

### 3. The app should feel comfortable for long study sessions

The UI must be clean, readable, responsive, and calm.

Users should be able to read, practice, and review without feeling overwhelmed.

### 4. The platform should prepare users for real work

Frontend lessons should focus on practical skills.

English lessons should focus on workplace communication, documentation, code reviews, interviews, and remote collaboration.

### 5. The product should support future expansion

The first version starts with Frontend Engineering and English for Tech Careers.

The product structure should support future tracks without major redesign.

### 6. MVP first, expansion later

The first version should be excellent in a small scope.

Do not build a huge platform too early.

## Language Strategy

FluentStack is built first for Indonesian learners.

The MVP learning content should use Bahasa Indonesia as the main explanation language.

Technical terms may remain in English when that is more natural for programming and technology learning.

Examples:
- semantic HTML
- component
- props
- state
- API
- deployment
- responsive design
- accessibility

English for Tech Careers should use a bilingual format.

Example structure:
- Indonesian explanation
- practical English sentence
- Indonesian meaning
- writing practice

The codebase, file names, components, routes, developer docs, and Codex prompts should stay in English.

This keeps the project professional for portfolio use while making the learning content easier for Indonesian users.

## Initial Product Experience

The user should be able to:

1. Open the landing page.
2. Understand what FluentStack is.
3. Start learning from the dashboard or roadmap.
4. Choose a track.
5. Open a module.
6. Open a lesson.
7. Read the lesson.
8. Complete quick checks, practice, and quiz blocks.
9. See progress increase.
10. Continue to the next lesson.

## Long-Term Product Vision

FluentStack can later become a broader technology learning platform with:

* more learning tracks
* user accounts
* saved progress
* project submissions
* AI-assisted feedback
* portfolio builder
* community features
* certificates
* career preparation
* interview practice

These features are not part of the first MVP unless explicitly requested.
